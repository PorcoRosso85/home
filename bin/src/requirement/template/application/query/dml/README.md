# データ操作言語（DML）

要件グラフを変更する操作群。

## 操作種別

### CREATE
新規ノード・エッジ作成。重複チェックと制約検証を含む。

### UPDATE
既存ノード属性更新。変更履歴の自動記録。

### DELETE
ノード・エッジ削除。論理削除による履歴保持。

## 主要テンプレート

- `create_requirement`: 要件作成（embedding自動生成）
- `create_dependency`: 依存関係作成（循環検証）
- `update_requirement`: 要件更新（バージョン追跡）

## 制約の自動適用

[ドメイン層](../../../domain/README.md)の不変条件を各操作で検証：
- 循環参照チェック
- グラフ深さ制限
- 一意性制約

## 上位概念

[クエリ体系](../README.md)の一部として、グラフ構造の変更を担当。
