# Review Directory

## 責務

このディレクトリは、Requirement Graph Systemのレビュー記録と再現可能なテストを管理します。

## ディレクトリ構造

```
review/
├── README.md          # このファイル
├── flake.nix         # Nixフレーク設定（将来の拡張用）
├── {yymmdd}.md       # レビュー結果ドキュメント
└── test_{yymmdd}.py  # レビュー時のテストを再現するスクリプト
```

## ファイル命名規則

- **レビュードキュメント**: `{yymmdd}.md`
  - 例: `250712.md` (2025年7月12日のレビュー)
  - 内容: ペルソナ別の使用感、良い点、改善点、総評

- **テストスクリプト**: `test_{yymmdd}.py`
  - 例: `test_250712.py`
  - 内容: レビュー時に実行したテストの再現可能なスクリプト

## 目的

1. **トレーサビリティ**: いつ、どのようなレビューが行われたかを追跡
2. **再現性**: 過去のレビューで発見された問題を再現可能に
3. **改善の追跡**: システムの進化と改善履歴を記録

## レビューの観点

各レビューでは以下の観点を評価します：

- **多様なペルソナ**: Owner、Manager、Engineerなど異なる立場からの評価
- **システム制約**: 循環依存、深さ制限、重複検出などの動作確認
- **ユーザビリティ**: 操作性、エラーメッセージの分かりやすさ
- **実用性**: 実際のプロジェクトでの適用可能性

## 使用方法

### レビューの実行

```bash
# テストスクリプトの実行
python review/test_250712.py
```

### 新しいレビューの追加

1. 新しいレビューを実施
2. 結果を`{yymmdd}.md`として保存
3. テストスクリプトを`test_{yymmdd}.py`として作成
4. このREADMEの更新は不要（命名規則に従えば自動的に整理される）
