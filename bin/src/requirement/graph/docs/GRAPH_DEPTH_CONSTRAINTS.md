# グラフ深さ制約

## 概要

RGLシステムでは、要件間の依存関係をグラフ構造で管理します。固定的な階層概念は存在せず、必要に応じてグラフのエッジ関係深さに上限を設けることができます。

## 基本原則

1. **階層は存在しない**: 要件に固定的な「レベル」や「階層」はありません
2. **柔軟な構造**: 任意の要件が任意の要件に依存可能です
3. **制約は任意**: プロジェクトのニーズに応じて、グラフの深さ制限を設定できます

## グラフ深さ制限の例

プロジェクトによっては、以下のような制限を設けることができます：

```cypher
-- 制限なし（デフォルト）
CREATE (a:RequirementEntity)-[:DEPENDS_ON]->(b:RequirementEntity)

-- 深さ制限の検証（アプリケーション層で実装）
-- 例：ある要件から5ホップ以上離れた要件への直接依存を避ける
```

## 重要な注意

- `hierarchy_level`、`level` などのプロパティは使用しません
- 「ビジョン」「タスク」などの名前は単なる例であり、システムによる特別な扱いはありません
- 制約はビジネスルールとして実装され、スキーマレベルでは強制されません
