# casting_board

メディア作成時のキャスティングの際に、不測の事態に事前対応するためのステートマシンをプロパティグラフデータベースで再現するシステム。

## 概要

メディア制作におけるキャスティングプロセスでは、様々な不測の事態が発生する可能性があります：
- キャスト候補者の急な辞退
- スケジュールの競合
- 予算制約の変更
- 役柄要件の変更

本システムは、これらの状況変化に柔軟に対応できるよう、プロパティグラフデータベースを用いてキャスティングの状態遷移と依存関係を管理します。

## 特徴

- **状態管理**: キャスト候補者とその状態（交渉中、確定、辞退等）をグラフ構造で表現
- **依存関係の可視化**: 役柄間の関係性（共演NG、必須ペア等）を明示的に管理
- **What-if分析**: 特定のキャストが辞退した場合の影響範囲を即座に把握
- **履歴管理**: キャスティング変更の経緯を時系列で追跡可能

## 技術スタック

- プロパティグラフデータベース（KuzuDB）
- ステートマシンによる状態遷移管理
- イベントソーシングによる履歴追跡

## 利用方法

```bash
# 開発環境のセットアップ
nix develop

# アプリケーションの実行
nix run .
```
