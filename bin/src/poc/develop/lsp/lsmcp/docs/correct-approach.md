# 正しいアプローチの検討

## 目標の再定義

### 当初の要求
「LSMCPをCLIとして使えるようにしたい（特にPython）」

### 問題点
1. LSMCPはMCPサーバーとして設計されている
2. CLI機能はTypeScript専用の実装
3. Python対応は不完全

## 取るべきアプローチ

### Option A: LSMCPの改善（本家にPR）
- PythonのCLIサポートを追加
- 診断機能の改善
- 時間がかかる、受け入れられるか不明

### Option B: LSMCPをforkして拡張
- 独自のPython CLI実装を追加
- 既存コードベースの理解が必要
- メンテナンスコスト高

### Option C: LSPを直接使うCLIツール作成
- pyright/pylspを直接呼び出す
- LSPプロトコルで通信
- 汎用的だが実装が複雑

### Option D: 実用的な代替ツール（現在の実装）
- Python専用ライブラリでCLI作成
- すぐ使える、動作確実
- LSPではないが目的は達成

## 推奨方針

### 短期的（現実的）
1. **現在の実装を整理・改善**
   - 不要なファイルを削除
   - ドキュメントを整理
   - 実用的なツールとして完成

### 中長期的（理想的）
2. **本当のLSP CLIツールを作成**
   - pyrightをCLIラッパーで包む
   - LSPプロトコル経由で通信
   - 他言語にも応用可能

## 次のステップ

1. 現在のディレクトリを整理
2. 実装したツールの位置づけを明確化
3. 必要に応じてLSP版も検討
