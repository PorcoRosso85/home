# SaaSパートナー市場調査 - マーケティング管理

## 責務
このモジュールは以下のマーケティング活動を管理します：
- コンタクトリスト管理（フォームURL、メールアドレス、SNS等）
- コンタクト時のスクリプトテンプレート管理
- 公式ドキュメント（../docs）との連携

## 市場調査方法

### 1. 資金調達データベースからのターゲット企業特定
**INITIALやSTARTUP DBを見て、シリーズA以降の資金調達をしたSaaS企業をリストアップしろ。100社はすぐに見つかる。**

- 対象: シリーズA以降の資金調達を実施したSaaS企業
- データソース: INITIAL、STARTUP DB等の資金調達データベース
- 期待結果: 100社以上のターゲット企業リスト

### 2. 採用ページからの痛み検出
**その企業の採用ページを見ろ。「事業開発」「BizDev」「パートナーサクセス」なんていう職種を募集していたら、そこはもう「痛み」のど真ん中だ。**

- チェックポイント:
  - 「事業開発」「BizDev」職種の募集
  - 「パートナーサクセス」「パートナーマネジメント」職種の募集
  - アライアンス関連職種の募集
- 判定: これらの職種募集 = パートナー管理の課題を抱えている証拠

### 3. SNSからの生の声収集
**Twitterで「SaaS パートナー」と検索しろ。悩みをツイートしている担当者や、イベントに登壇している責任者がゴロゴロ出てくる。**

- 検索キーワード:
  - 「SaaS パートナー」
  - 「パートナープログラム 管理」
  - 「代理店管理 SaaS」
- 収集対象:
  - 課題をツイートしている担当者
  - イベント登壇者・責任者
  - ディスカッションに参加している実務者

### 4. イベントでの直接検証
**SaaS系のオンラインイベントに参加しろ。ネットワーキングの時間に、「パートナープログラムって、皆さんどうやって管理してます？」と聞いてみろ。蜘蛛の子を散らすように人が集まってくるか、全員が天を仰ぐかのどっちかだ。**

- 参加対象イベント:
  - SaaS業界カンファレンス
  - BizDev系ミートアップ
  - パートナー戦略セミナー
- 質問例:
  - 「パートナープログラムって、皆さんどうやって管理してます？」
  - 「代理店の成果測定ってどうされてます？」
  - 「パートナー向けの情報共有ってどんなツール使ってます？」
- 期待される反応:
  - 強い関心（人が集まる） = 共通の痛み
  - 困惑・苦笑い（天を仰ぐ） = 解決策がない痛み

## コンタクトリスト構造

```yaml
contacts:
  - company: 企業名
    stage: シリーズA/B/C等
    pain_signals:
      - hiring: [募集職種]
      - social: [SNS投稿URL]
      - event: [イベント参加情報]
    contact_points:
      - type: form/email/sns
        url: コンタクトURL
        person: 担当者名（判明している場合）
    priority: high/medium/low
    notes: 特記事項
```

### 初回コンタクト用
```

貴社が{pain_signal}されているのを拝見し、
パートナープログラム管理に課題をお持ちかと思いご連絡しました。

弊社では、SaaSパートナー管理を効率化する
ソリューションを提供しており...
```

実は同様の課題を抱える企業様向けに...
