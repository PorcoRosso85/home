# Presentation Layer UI要件仕様書

## 基本方針
UIの唯一の目的: **ユーザーの思考を一切邪魔せず、迷わせず、自己完結させること**

## UI実装の掟三箇条

### 1. レイアウトは「聖域」として一切いじるな

**画面構成（固定）:**
- 左半分: 入力フォーム
- 右半分: 出力結果（グラフと数値）
- ヘッダー: ロゴとログアウトボタンのみ
- フッター: 不要

**理由:** ユーザーの視線移動を最小化し、入力と結果の因果関係を一瞬で理解させる

### 2. 使う色は「3色」まで

**配色ルール:**
- 背景色: 白 or 目に優しいダークグレー
- 文字色: 黒 or 白
- アクセントカラー: 青などの原色を1色のみ

**使用原則:** 
- アクセントカラーは重要箇所（エラー表示、結果強調）にのみ使用
- 装飾的な色使いは厳禁

### 3. コンポーネントは「標準部品」以外使うな

**許可される部品:**
- テキストボックス
- ドロップダウンリスト
- チェックボックス
- ラジオボタン
- 普通のボタン

**理由:** 全ユーザーが既に使い方を知っている = 学習コストゼロ

## 技術的制約

### CSS制約
- **Vanilla CSSのみ使用**（CSSフレームワーク使用禁止）
- 機能に関係ないCSSは1行も書かない
- 装飾的なスタイリングは厳禁

### ディレクトリ構造
```
presentation/
├── README.md        # 本ファイル
├── index.tsx        # Reactマウントポイント
├── App.tsx          # メインアプリケーションコンポーネント
├── components/      # UIコンポーネント（標準部品のみ）
├── pages/          # ページコンポーネント
└── styles/         # 最小限のCSS
    └── index.css   # グリッドとレイアウトのみ
```

## ビジネス上の原則

### マニュアル不要が最高の信頼性
- 標準部品のみ = 直感的操作
- チュートリアル不要 = 即座に使い始められる
- これがB2B SaaSにおける信頼性の証明

### 一貫性による効率化
- 全画面で統一されたルール
- 開発者は「ルール通りに作るだけ」
- UI設計時間ゼロ = 利益率向上

### プロフェッショナルツールの外観
- シンプル ≠ 安っぽい
- 要素の整列と適切な余白
- 装飾の排除 = 「中身で勝負」の自信

## 実装チェックリスト

- [ ] 3色以内の配色

- [ ] マニュアルなしで操作可能か確認

## 最終結論
最小限のUI要件とは、**「ユーザーに何も考えさせず、本来の目的である『計算とシミュレーション』に100%集中させるための、徹底的にノイズを排除した環境」**のことである。

それ以上でも、それ以下でもない。
