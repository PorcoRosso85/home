Phase 1.3 — Edge-Split（tracker / sitemap / edge-cache に分割）

目的
- 1.2 の単一 Worker を役割ごとに分離し、スケール/保守/障害分離を強化する。

スコープ
- interfaces/tracker（ingest/redirect）
- interfaces/sitemap（/sitemap.xml, /sitemap-[n].xml, /ping）
- interfaces/edge-cache（/revalidate と標準ヘッダ付与）
- jobs/edge_cache_prime.ts

作業ツリー（作成）
- interfaces/tracker/{src/ingest.ts,src/redirect.ts,wrangler.toml}
- interfaces/sitemap/{src/index.ts,src/segment.ts,src/ping.ts,wrangler.toml}
- interfaces/edge-cache/{src/revalidate.ts,src/headers.ts,wrangler.toml}
- jobs/edge_cache_prime.ts

実装チェックリスト
- [ ] wrangler.shared.toml を extends し、最小権限で各 Worker を構成
- [ ] sitemap: 5万URL/ファイルの分割（小規模は単一で可）と lastmod 設定
- [ ] edge-cache: Cache-Control, ETag, Vary の標準化、/revalidate でプリウォーム
- [ ] tracker: ingest/redirect を移設（edge-min を停止）

受け入れ基準（DoD）
- [ ] 3 Worker が個別にデプロイ成功
- [ ] site（rwsdk-init）との併用で全体の疎通が保たれる
- [ ] 既存 URL のキャッシュを /revalidate で更新可能

ロールバック
- いずれかの Worker 障害時も他は稼働。必要なら edge-min に一時的に戻す。
