# Vessel - 動的スクリプト実行コンテナ

## 概要

Vesselは、動的なスクリプトを安全に実行するための「器」です。標準入力からスクリプトを受け取り、テンプレート内で定義された実行環境で動作します。環境変数に依存せず、純粋にテンプレートファイルと入力スクリプトのみで動作します。

## 責務

1. **動的な値としてのスクリプトを代入できる器**
   - 標準入力からスクリプトを受け取る
   - 実行コンテキストを提供する
   - データと処理を分離する

2. **器の器**
   - 再利用可能なテンプレートとして機能
   - 様々なユースケースに対応可能
   - 拡張性を持つ設計

## 基本的な使い方

### シンプルな例

```bash
# 標準入力でロジックを注入
echo 'print("Hello from vessel!")' | python vessel.py
```

### ファイルからスクリプトを読み込む例

```bash
# スクリプトファイルを作成
cat > script.py << 'EOF'
for i in range(5):
    print(f"Count: {i}")
EOF

# vesselで実行
python vessel.py < script.py
```

### パイプラインでの使用例

```bash
# 他のコマンドの出力をスクリプトとして実行
echo 'print(sum([1, 2, 3, 4, 5]))' | python vessel.py
```

## ガードレール

- 入力検証
- エラーハンドリング
- 安全な実行環境の提供

## ゲームチェンジャーとしての価値

1. **環境変数に依存しない純粋な設計**
   - テンプレートと入力スクリプトのみに依存
   - 予測可能で再現性の高い動作

2. **責務の明確な分離**
   - Vessel: 実行環境の提供
   - 入力スクリプト: ビジネスロジック
   - 標準入出力: インターフェース

3. **拡張性**
   - 異なるテンプレートの作成が容易
   - パイプライン処理との親和性
