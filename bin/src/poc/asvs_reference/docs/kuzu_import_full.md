# KuzuDB Import Documentation (要約版)

**注意**: WebFetchツールの制限により、完全なドキュメントではなく要約が取得されました。
正確な情報は公式ドキュメント（https://docs.kuzudb.com/import/）を直接参照してください。

## 取得できた主要情報

### インポート方法の概要
- 小規模グラフ: `CREATE`と`MERGE` Cypherクローズを使用
- 大規模グラフ（数百万ノード）: `COPY FROM`を推奨

### サポートされるインポート形式
- CSV
- Parquet  
- NumPy
- Subquery
- DataFrame
- JSON

### バルクインサートのシナリオ
- カラムのサブセットからの挿入
- 複数のFROM-TO接続を持つリレーションシップテーブル
- 大規模リレーションシップテーブルのインポート

### エラーハンドリング
- `IGNORE_ERRORS=true`を使用してエラー行をスキップ可能
- スキップされた行を追跡するWarningsテーブル
- 警告制限の設定可能

### データソース別のスキップ可能なエラー
CSVが最も柔軟で、JSON/Parquetはより厳格なエラーハンドリング
