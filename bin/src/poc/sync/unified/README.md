# KuzuDB Unified Sync POC

## 責務

このディレクトリは以下の責務を持つ：

- **複数のKuzuDBインスタンスでの同期可能性再確認**
  - 複数のKuzuDBインスタンス間でのデータ同期メカニズムの検証
  - 同期の整合性と信頼性の確認

- **本番環境での使用を想定**
  - 本番環境での実用性を前提とした設計と実装
  - エラーハンドリングとリカバリー機能の実装

- **マイグレーション可能性**
  - 既存システムからの移行パスの検証
  - データ移行時の整合性保証

- **パフォーマンステスト済み**
  - 同期処理のパフォーマンス検証完了
  - 本番環境で期待される負荷での動作確認

## テスト実行方法

以下のコマンドでE2E/統合テストを実行できます：

```bash
nix run .#test
```

### テスト構成

- **E2Eテスト**: Python/pytest (`tests/e2e_test.py`)
  - KuzuDBインスタンス間の同期機能の完全な動作検証
  - エンドツーエンドのシナリオテスト

- **統合テスト**: TypeScript/Deno (`tests/integration.test.ts`)
  - コンポーネント間の連携テスト
  - API統合テスト
