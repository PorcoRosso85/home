# テスト方針

## 目的

各関数の単体テストは、その関数が「何を目的としているか」を明確に示すものであるべきです。テストは単なる技術的な検証ではなく、関数の仕様や使用例を例示する役割も担います。

## テスト追加の原則

1. **正常系テストを優先**：各関数に対して、少なくとも1つの正常系テストを追加します
2. **仕様の明示**：テストは関数が何を意図しているのかが明確に伝わるようにします
3. **使用例の例示**：テストは同時に関数の使用例としても機能するよう設計します
4. **簡潔性**：テストは短く簡潔に記述します
5. **関数型スタイル**：クラスではなく関数を使用します

## テスト対象ファイル

1. **cliUtils.ts**：CLI関連のユーティリティ関数群に対するテスト
2. **dependencyAnalyzer.ts**：依存関係解析の中核機能に対するテスト
3. **referenceResolver.ts**：スキーマ参照の解決機能に対するテスト

## テスト実行方法

テストはファイル内に直接含まれ、そのファイルを実行するだけでテストが実行されます。
```
./ファイル名.ts
```
