# ドキュメント

## 基本方針

コードは、未来の自分や他の開発者への手紙です。ドキュメントは、コードの意図を補完し、再利用を促進するために不可欠です。

## 要求事項

- **公開関数にはドキュメント必須**: `mod.{ext}` からエクスポートされるすべての関数や型には、ドキュメントを記述しなければなりません。
- **ドキュメントに含める内容**:
  1.  **概要**: その関数や型が何をするのか、簡潔に説明します。
  2.  **引数**: 各引数の意味と、期待される型や値を説明します。
  3.  **戻り値**: 関数が何を返すのか、成功時と失敗時の両方について説明します。
  4.  **使用例**: 実際にその関数をどのように使うのか、実行可能なコードスニペットを必ず含めます。
- **使用例は `mod.{ext}` に記載**: モジュールの使い方を示す総合的な使用例は、`README.md` ではなく、`mod.{ext}` のドキュメントコメント内に記載します。これにより、コードとドキュメントが常に同期します。

## フォーマット

各言語で標準的に使われるドキュメンテーションコメントのフォーマットに従います。

| 言語 | フォーマット |
| :--- | :--- |
| **TypeScript** | JSDoc (`/** ... */`) |
| **Python** | Docstring (`""" ... """`) |
| **Go** | ゴーコメント (`// ...`) |
| **Rust** | ドキュメントコメント (`/// ...`) |
| **Zig** | ドキュメントコメント (`/// ...`) |
