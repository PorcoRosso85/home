# 設計原則

- **データと処理の分離**: オブジェクトから振る舞いをなくし、データ構造とそれを操作する関数を明確に分離します。
- **エラーを値として扱う**: 例外を投げるのではなく、エラーを関数の戻り値として返し、呼び出し元で明示的に処理させます。
- **依存性を明示的に注入**: 必要な依存関係は、関数の引数として外部から渡します。グローバルな状態や暗黙の依存を排除します。
- **最小構成を維持**: 必要最小限の機能だけを実装し、過剰な抽象化や将来の予測に基づく設計を避けます。
- **関数の合成可能性を重視**: 小さく、純粋な関数を組み合わせることで、複雑な機能を実現します。
- **副作用を境界に隔離**: ファイルI/O、ネットワーク通信、データベースアクセスなどの副作用を伴う処理は、アプリケーションのコアロジックから分離し、特定のレイヤー（インフラストラクチャ層など）に限定します。
