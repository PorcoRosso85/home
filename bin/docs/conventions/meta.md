# 規約のための規約

## 前提

これらの規約は、高品質で保守性の高いコードベースを維持するためのものです。
しかし、規約が形骸化しては意味がありません。

## Linter/Formatter設定の義務

プロジェクトを開始する前、または新しい言語を導入する際には、必ず対応するLinterおよびFormatterの設定ファイルが存在することを確認してください。

- **設定ファイルが存在しない場合**: 新しく作成し、基本的なルール（命名規則、禁止事項など）を定義してから開発に着手してください。
- **アラート**: 設定が存在しない状態で開発を進めることは、技術的負債を生む原因となります。直ちにチームにアラートし、設定を整備してください。

このルールは、規約の遵守を個人の努力だけに依存させず、ツールによって自動的に担保するためのものです。

## 規約ドキュメントの構成

1. **原則** - その規約が目指す理想的な状態
2. **ルール** - 具体的な実装規則
3. **実装パターン** - 言語別の具体例
4. **アンチパターン** - 避けるべき実装
5. **ベストプラクティス** - 推奨される実装方法

新しい規約を追加する際は、この構成に従ってください。
