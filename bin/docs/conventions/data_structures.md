# データ構造

## 原則

- **不変性（Immutability）を優先**: データは一度作成したら変更しないことを原則とします。変更が必要な場合は、新しいデータを作成します。
- **振る舞いを持たないデータ構造**: データ構造は状態を保持するだけで、メソッド（振る舞い）を持たないようにします。データと処理を完全に分離します。
- **ハードコード値の禁止**: マジックナンバーや文字列リテラルを直接コードに埋め込まず、型定義や定数として名前を付けて管理します。

## 許可されるデータ構造

| 言語 | 許可される形式 |
| :--- | :--- |
| **TypeScript** | `type` （`interface`は禁止）, `enum` |
| **Python** | `TypedDict`, `dataclass(frozen=True)` |
| **Go** | `struct` |
| **Rust** | `struct`, `enum` |
| **Zig** | `struct`, `enum` |

**Note**: `interface` の禁止理由は、`type` がより柔軟で、合併型（Union Types）や交差型（Intersection Types）との親和性が高いためです。
