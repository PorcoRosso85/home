# /conventions
/conventions

# 説明
現在の文脈に適した規約を確認し、タスクに統合する

# 実行内容
1. 現在のタスクに関連する規約をイメージする
2. `/home/nixos/bin/docs/conventions/`から該当規約を確認
3. 規約と現行タスクのすり合わせを脳内で実施
4. すり合わせ完了後、タスクを規約準拠で整理
5. 必要に応じて実装を修正

# 規約確認の流れ
```
現在のタスク分析
    ↓
関連規約の特定
    ↓
規約ファイル読み込み
    ↓
タスクとのすり合わせ
    ↓
規約準拠のタスク整理
```

# 主要な規約カテゴリ
- **error_handling.md**: エラー処理とフォールバック
- **commit_messages.md**: コミットメッセージ形式
- **tdd_process.md**: TDD実践方法
- **module_design.md**: モジュール設計原則
- **security.md**: セキュリティとシークレット管理
- **logging.md**: ロギング規約
- **testing.md**: テスト設計と実装

# 実行例
```
ユーザー: エラー処理を実装して

Claude: /conventions を実行します。

1. 関連規約のイメージ：error_handling.md
2. 規約確認中...
   - フォールバック禁止
   - エラーは明示的に報告
   - 早期リターン推奨
3. 現行タスクとのすり合わせ完了
4. 規約準拠で実装を進めます
```

# 注意事項
- 規約は絶対的なルールとして扱う
- 規約に違反する既存コードは修正対象
- 不明な場合は複数の規約を確認
