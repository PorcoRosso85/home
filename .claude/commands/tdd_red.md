# tdd_red
/tdd_red

# 説明
t-wada流TDDのRedフェーズを規約に則って実行し、失敗するテストを作成する

# 実行内容
1. 規約ファイルの場所を確認
   - ユーザーに規約ファイルのパスを問い合わせる
   - デフォルト候補: ~/bin/src/CONVENTION.yaml
2. 規約ファイルを読み込み、TDDに関する規定を確認
3. 実装予定の機能について確認
4. 規約に従って失敗するテストを作成
   - テストファイルの命名規則に従う
   - テストの構造やアサーションスタイルを規約に合わせる
5. テストを実行して失敗することを確認
6. 失敗内容を報告

# TDD Redフェーズの原則
- まだ存在しない機能のテストを書く
- テストは必ず失敗する（コンパイルエラーも含む）
- 最小限のテストから始める
- 期待する振る舞いを明確に表現する

# 注意事項
- 規約ファイルの指定がない場合は必ず問い合わせる
- テストの失敗を確認してからRedフェーズ完了とする
