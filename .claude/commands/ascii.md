# ascii
/ascii

# 説明
ASCII形式のシーケンス図を生成して、処理の流れや相互作用を視覚化する

# 実行内容
1. 対象となるコードや処理の流れを分析
2. 主要なアクター/コンポーネントを特定
3. メッセージの流れと相互作用を整理
4. ASCII文字を使用してシーケンス図を作成
5. 図に説明や注釈を追加

# 図の形式
```
┌─────────┐          ┌─────────┐          ┌─────────┐
│ Actor A │          │ Actor B │          │ Actor C │
└────┬────┘          └────┬────┘          └────┬────┘
     │                     │                     │
     │  1. Request         │                     │
     ├────────────────────>│                     │
     │                     │  2. Process         │
     │                     ├────────────────────>│
     │                     │                     │
     │                     │  3. Response        │
     │                     │<────────────────────┤
     │  4. Result          │                     │
     │<────────────────────┤                     │
     │                     │                     │
```

# 使用例
- APIの呼び出しフロー
- データベースのトランザクション処理
- マイクロサービス間の通信
- イベント駆動アーキテクチャの流れ
- 認証・認可のプロセス
