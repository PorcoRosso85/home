# /review_by_user
/review_by_user

# 説明
ユーザー視点でシステムをレビューし、使いやすさと目的の達成度を確認する

# 実行内容
1. 日本語でユーザーと対話する
2. ユーザーの目的・意図を確認する
3. このシステムを初めて使うユーザーを想定して説明
4. ユーザーが本当に欲しいものは何かを探る
5. システムの現状をユーザー視点でレビューする

# レビューポイント
- **目的の明確化**: ユーザーは何を達成したいのか？
- **使いやすさ**: 初見のユーザーでも理解できるか？
- **期待と現実**: システムは期待に応えているか？
- **改善点**: どこを改善すべきか？

# 実行例
```
ユーザー: /review_by_user

Claude: このシステムについて、ユーザー視点でレビューさせていただきます。

まず、このシステムで何を達成したいと考えていらっしゃいますか？
例えば：
- 開発作業の効率化
- 複数タスクの並列処理
- エラー解決の自動化
- その他の目的

現在のシステムの使い勝手はいかがでしょうか？
初めて使う方でも理解しやすい設計になっているでしょうか？
```

# 注意事項
- 技術的な詳細よりも、ユーザーの体験に焦点を当てる
- 専門用語を避け、分かりやすい日本語で説明
- ユーザーの本音を引き出すよう心がける
