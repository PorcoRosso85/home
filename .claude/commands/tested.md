# tested
/tested

# 説明
テストとアプリケーションの整合性確認

# 実行内容
0. 禁止事項確認: `bin/docs/conventions/prohibited_items.md`

1. **実行ディレクトリの責務・目的確認**
   - 現在のディレクトリのREADMEを確認
   - このディレクトリの目的・責務を明確に宣言
   - テスト評価の文脈と制約を理解

2. **テスト実行の判定**
   - 直近でテストが実行済みの場合は省略可能
   - 未実行またはコード変更がある場合は実行
   - ユーザーが目的確認のみを依頼した場合はそれに従う

3. **仕様としての存在意義評価**
   - テストが仕様を正確に表現しているか確認
   - アプリケーション要件への貢献度を評価
   - 責務・目的に照らした妥当性の検証

4. **規約準拠の確認**
   - 整合性確認の観点は `/bin/docs/conventions/testing.md#テストとアプリケーションの整合性確認` を参照
   - テスト実行環境の確認は `/bin/docs/conventions/test_infrastructure.md` を参照
   - 必要に応じて `nix run .#test` でテスト実行

# 実行モード
- **フル実行**: すべてのステップを実行（デフォルト）
- **目的確認のみ**: ステップ1のみ実行（ユーザー指定時）
- **テスト省略**: ステップ1,3,4を実行（直近テスト済みの場合）
