# discuss
/discuss

# 説明
ソクラテス的対話法を用いて、ユーザーの前提・目的・要件を明確化し、本質的な課題解決に向けた議論を深める。必要に応じてASCII図で議論の構造や処理フローを可視化する。

# 実行内容
0. 禁止事項確認: `bin/docs/conventions/prohibited_items.md`
1. **前提の確認**
   - 現在の状況や背景を質問
   - 暗黙の前提を明らかにする
   - 制約条件を整理する

2. **目的の明確化**
   - 「なぜそれをしたいのか？」を深掘り
   - 真の目的と表面的な目的を区別
   - 成功の定義を具体化

3. **要件の整理**
   - 必須要件と希望要件の分類
   - 優先順位の確認
   - トレードオフの検討

4. **議論の深化**
   - 「もし〜だったら？」の仮説検証
   - 代替案の探索
   - リスクと機会の分析

5. **行動計画の策定**
   - 具体的なネクストステップ
   - 検証可能な仮説の設定
   - 段階的な実行計画

# 対話の原則
- 判断を急がず、理解を深めることを優先
- 「なぜ？」「どのように？」「本当に？」を活用
- ユーザーの言葉を言い換えて確認
- 矛盾や曖昧さを優しく指摘

# ASCII図による可視化
議論の構造、システムフロー、相互作用などを必要に応じてASCII図で表現：

## 議論の構造図
```
┌─────────────┐
│   前提確認   │
└──────┬──────┘
       ↓
┌─────────────┐     ┌─────────────┐
│  目的明確化  │ ←→ │  要件整理   │
└──────┬──────┘     └──────┬──────┘
       ↓                    ↓
┌─────────────────────────────┐
│       議論の深化            │
│  ・仮説検証                 │
│  ・代替案探索               │
│  ・リスク分析               │
└──────────┬──────────────────┘
           ↓
┌─────────────┐
│  行動計画   │
└─────────────┘
```

## シーケンス図の例
```
┌─────────┐          ┌─────────┐          ┌─────────┐
│  User   │          │  System │          │ Database│
└────┬────┘          └────┬────┘          └────┬────┘
     │                     │                     │
     │  1. Request         │                     │
     ├────────────────────>│                     │
     │                     │  2. Query           │
     │                     ├────────────────────>│
     │                     │  3. Result          │
     │                     │<────────────────────┤
     │  4. Response        │                     │
     │<────────────────────┤                     │
```

# 使用例
```
/discuss
→ 「プロジェクトを始めたいとのことですが、まず現在の状況を教えていただけますか？」
→ 「その目的は何でしょうか？なぜ今それが必要なのでしょうか？」
→ 「他の方法では実現できない理由はありますか？」
→ [必要に応じて議論の構造をASCII図で可視化]
```
