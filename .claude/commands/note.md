# note
/note

# 説明
直近の議論・指示の一貫性を判断し、再利用可能なドキュメントの作成を支援する

# 実行内容
0. 禁止事項確認: `bin/docs/conventions/prohibited_items.md`
1. **議論の一貫性分析**
   - 直近の会話履歴を分析
   - トピックの一貫性を評価
   - 再利用可能な知見の抽出

2. **ドキュメント化の提案**
   - **一貫している場合**: 
     - 主題を自動識別
     - 保存場所の提案（docs/, conventions/, README等）
     - **meta.md準拠の構造提案**（規約の場合は7つの構成要素）
   - **一貫していない場合**:
     - ユーザーに主題の確認
     - 複数の主題がある場合は分割提案

3. **ドキュメント作成**
   - 承認後、適切な形式でドキュメント化
   - **meta.mdの構成に従った文書作成**（規約の場合）
   - **既存ドキュメントとの相互参照を追加**
   - 必要に応じて関連ファイルの更新

# ドキュメント構成（規約作成時）
`bin/docs/conventions/meta.md`に準拠：
1. **原則** - その規約が目指す理想的な状態
2. **ルール** - 具体的な実装規則
3. **実装パターン** - 言語別の具体例
4. **アンチパターン** - 避けるべき実装
5. **ベストプラクティス** - 推奨される実装方法
6. **関連規約** - 他の規約との相互参照
7. **理由** - なぜこの規約が必要か

# 保存先の基準
- **規約・ルール**: `bin/docs/conventions/`
- **プロジェクト固有**: 各プロジェクトのdocs/
- **全体的な知見**: プロジェクトルートのドキュメント
- **コマンド関連**: `~/.claude/commands/`の更新

# 実行例
```
ユーザー: /note
Claude: 直近の議論を分析しました。
「作業範囲規則の強調」について一貫した議論がされています。
これをconventions/work_scope_rules.mdとして文書化することを提案します。
よろしいですか？

ユーザー: はい
Claude: [ドキュメントを作成]
```
