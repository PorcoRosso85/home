# spec
/spec

# 説明
プロジェクトのflake.nixを確認してテストフレームワークを特定し、テストファイルを探索することでディレクトリの責務が仕様としてテストに表現されているか確認する

# 実行内容
1. 対象ディレクトリパスを確認（不明な場合はユーザーに確認）
2. プロジェクトのflake.nixを読み込んでテストフレームワークを特定
3. `nix develop --command pytest --collect-only` または同等のコマンドでテスト一覧を取得
4. テスト名と説明から仕様を抽出して整理
5. ディレクトリの責務とテストの対応関係を分析
6. 仕様として不足している部分があれば指摘

# 使用例
```
/spec
→ 現在のディレクトリで仕様確認

/spec /home/nixos/bin/src/poc/kuzu_migration
→ 特定のディレクトリで仕様確認
```

# 出力形式
- テストフレームワーク: pytest/jest/go test等
- テスト総数: N個
- モジュール別の仕様一覧
- 責務とテストの対応関係
- 仕様の充足度評価
