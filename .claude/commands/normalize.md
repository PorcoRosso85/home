# normalize
/normalize

# 説明
対象に対して責務分離・正規化・可逆圧縮を徹底的に実施する。

# 実行内容
0. 禁止事項確認: `bin/docs/conventions/prohibited_items.md`
1. **作業ディレクトリの目的確認**
   - 現在のディレクトリのREADMEを確認
   - このディレクトリの目的・責務を明確に宣言
   - 正規化の文脈と制約を理解

2. **正規化の実施**
   対象に対し以下を徹底する：
   - **責務分離化**: 各要素が単一の明確な責務のみを持つ（SOLID原則のS）
   - **責務正規化**: 重複を完全に排除し、単一の真実の源を確立（DRY原則）
   - **可逆圧縮化**: デグレ一切無く文章・コードをコンパクトに（KISS/YAGNI原則）

# 対象の決定
対象はユーザーからの指定がない場合、事前の文脈からユーザーへの対象提案を行い確定を受け取る。

# 実行手順
1. 作業ディレクトリのREADME確認と目的宣言
2. 対象の確認・提案
3. 現状分析（責務の重複、冗長性の特定）
4. 正規化案の提示
5. ユーザー承認後に適用

# 使用例
```
ユーザー: /normalize
Claude: 直前の文脈から以下を対象として提案します：
- entry.md, nix_flake.md, module_design.md の相互関係
正規化を実施してよろしいですか？

ユーザー: はい
Claude: [正規化を実施]
```

# 原則
- 情報の欠落なし（可逆性の保証）
- 参照関係の明確化
- 70%以下への圧縮を目標
