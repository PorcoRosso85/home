# コマンド名
/propose

# 説明
文脈に沿った提案を行うコマンド。ユーザーの反論を想定し、具体的で実現可能な提案を生成する。

# 実行内容
1. **文脈確認**
   - 会話履歴から要件・制約を抽出
   - conditions.mdとconstraints.mdがあれば参照
   - ユーザーの真のニーズを理解

2. **提案生成と自己検証**
   - 3つ以上の提案候補を内部で生成
   - 各提案に対して自己反論を実施
   - 反論に耐えられる提案のみ選定

3. **反論想定と回答準備**
   - 予想される反論を3つ以上リストアップ
   - 各反論への具体的回答を用意
   - 残る課題も正直に提示

4. **具体化（APE適用）**
   - 抽象的な提案は避ける
   - 実装手順・初期ステップを明確化
   - 数値・期間・規模を具体的に示す

# 出力形式
```
## 提案：[提案タイトル]

### 概要
[1-2文で提案を要約]

### 具体的な仕組み
[図や箇条書きで説明]

### 予想される反論と回答
1. **反論**: [内容]
   **回答**: [具体的な対応策]

### 初期ステップ（最初の2週間）
- Day 1-3: [具体的なアクション]
- Day 4-7: [具体的なアクション]

### 残る課題
- [正直な課題1]
- [正直な課題2]
```
